# 星云书法学习应用体验重构设计文档

版本：V1.1  
日期：2026-06-30  
适用产品：nebula-calligraphy Web / 移动端 MVP  
目标阶段：生产试用前体验重构

V1.1 变更说明：吸收外部体验评审意见，将“像学习软件”进一步落到具体视觉、文案、交互反馈和验收规则上。此版本作为下一轮前端重构的执行依据。

## 1. 背景与问题诊断

当前版本已经具备注册登录、查字、临摹记录、常用字、章法预览和导出等基础能力，但前端体验仍然像“功能演示页”，不是一个真正面向书法学习者的应用。

主要问题：

- 学习目标不明确：用户登录后不知道今天应该练什么、先看什么、下一步做什么。
- 临摹参考不突出：书法学习的核心是“看帖、读帖、临帖”，当前参考字没有成为视觉中心。
- 笔法、结构、章法割裂：基本笔画、单字间架结构、多字章法没有形成递进路径。
- 功能入口像后台工具：查字、集字、作品等功能并列摆放，缺少学习任务驱动。
- 缺少主流学习软件的节奏：没有“今日任务、课程路径、练习反馈、进度沉淀”的闭环。
- 视觉层级不专业：统计、按钮、卡片权重接近，导致重点被稀释。
- 缺少宣纸和字帖氛围：纯白背景和通用卡片让应用显得像企业后台。
- 参考字尺寸不足：临摹对象没有成为主视觉中心。
- 参考字字体缺少书法韵味：默认系统字体无法激发临摹欲望。
- 内部字段暴露给用户：例如 `common-practice-ou` 这类技术 ID 不应出现在界面。
- 操作反馈不足：记录临摹、生成章法、保存草稿后缺少即时反馈。

结论：下一版不应继续微调布局，而应按“书法学习工作台”重构信息架构和核心页面。

## 2. 产品定位

星云书法不是普通字典，也不是单纯集字工具。它应定位为：

> 一个以临摹训练为中心，融合查字、笔法解析、结构分析和章法创作的书法学习工作台。

核心使用场景：

- 初学者每天临一个字，理解基本笔画和结构。
- 进阶用户查名家写法，比较书体差异。
- 创作者将多个字组织成作品章法。
- 教师或学习者生成临摹参考和作品草稿。

## 3. 设计原则

### 3.1 临摹对象优先

用户进入应用后，第一眼必须看到今天要临的参考字。参考字要足够大、足够安静、足够像字帖，而不是小卡片里的一个普通字符。

### 3.2 学习路径优先于功能入口

主页面不应先展示“查字、创作、作品”等工具入口，而应展示学习路径：

1. 看参考字。
2. 拆基本笔画。
3. 看单字结构。
4. 做临摹练习。
5. 进入多字章法。

工具入口应服务于路径，而不是取代路径。

### 3.3 笔法、结构、章法三层递进

书法学习不是只查一个字。页面必须同时承载三类知识：

- 基本笔画：横、竖、撇、捺、点、折、钩，关注起笔、行笔、收笔。
- 单字结构：中宫、主笔、重心、收放、避让、穿插。
- 多字章法：行列、字距、行气、留白、落款、钤印。

### 3.4 少解释，多操作

页面不应堆大段说明文字。每个区域都应该有明确动作：

- 换一个字。
- 看笔法。
- 看结构。
- 开始临摹。
- 生成章法。
- 保存作品。

### 3.5 MVP 允许无真实碑帖图片，但不能无参考

如果当前还没有授权碑帖图片，也必须提供高质量的替代参考：

- 米字格。
- 大号参考字。
- 书体、书家、碑帖来源。
- 笔画拆解。
- 结构辅助线。

后续接入真实碑帖图片后，再替换参考字渲染层。

### 3.6 视觉必须像学习工具，不像后台

书法应用的第一印象应接近“摊开的练习纸”和“清晰的课程任务”，而不是数据后台。背景、卡片、按钮和字号必须服务于长时间学习。

硬性要求：

- 全局背景使用暖灰白：`#F7F4EB` 或 `#F5F0E6`。
- 卡片背景使用 `#FFFFFF`，阴影控制在 `0 2px 8px rgba(0,0,0,0.04)`。
- 减少浮动卡片感，避免卡片套卡片。
- 主按钮使用深绿或墨赭实心色，页面内只保留一个最强主操作。
- 参考字字体优先使用 `"华文楷体", "KaiTi", "STKaiti", serif`。

### 3.7 用户行为必须有反馈

任何学习动作都必须给反馈。无反馈会让用户误以为功能坏了。

最低要求：

- 记录临摹后显示“已记录 ✓”。
- 今日练习次数立即 +1。
- 生成章法后在原预览区直接显示结果。
- 保存草稿后显示“已保存”，并允许跳转作品页。
- 点击笔画或结构标签后，必须有选中态和说明变化。

### 3.8 降低首次使用成本

学习软件不能假设用户理解功能结构。首次进入应给出轻量路径，而不是展示一堆入口。

最低要求：

- 今日页展示 3 步引导：`选今日字 → 看临摹参考 → 我已临摹`。
- 默认加载今日字，避免空白首页。
- 明确提供“一键换字”。
- 操作文案使用学习语言，而非系统语言。
- 错误提示必须面向用户，不暴露 `ApiException`、`http_error` 等内部信息。

## 4. 目标用户路径

### 4.1 每日练习路径

用户打开应用后进入“今日练习”。

流程：

1. 系统默认给出一个今日字，例如“永”。
2. 页面中心展示大号临摹参考。
3. 用户查看基本笔画拆解。
4. 用户查看结构分析。
5. 用户点击“开始临摹”或“记录一次临摹”。
6. 系统记录练习，并更新学习进度。
7. 用户可切换到下一个推荐字。

### 4.2 查字学习路径

用户有明确想学的字。

流程：

1. 输入单字。
2. 选择书体。
3. 查看多个候选写法。
4. 进入单字详情。
5. 单字详情展示参考字、笔法、结构、临摹模板。
6. 用户加入今日练习或记录临摹。

### 4.3 章法创作路径

用户想写一首诗或一段文字。

流程：

1. 输入诗文。
2. 选择书体。
3. 选择幅式。
4. 生成章法预览。
5. 调整纸张、字距、落款。
6. 保存草稿。
7. 导出参考 SVG / 后续 PDF。

## 5. 信息架构

建议主导航从“功能并列”调整为“学习路径 + 工具补充”。

底部导航：

1. 今日
2. 查字
3. 创作
4. 作品

其中“今日”必须是默认页，承担学习闭环。

### 今日页

核心职责：让用户知道今天练什么、怎么练。

区域顺序：

1. 今日任务头部。
2. 临摹参考主卡。
3. 三段训练路径。
4. 推荐字队列。
5. 学习进度摘要。

### 查字页

核心职责：快速查询名家写法，并进入临摹。

区域顺序：

1. 搜索框。
2. 书体筛选。
3. 结果列表。
4. 单字详情。

### 创作页

核心职责：从文字到章法参考。

区域顺序：

1. 文本输入。
2. 书体和碑帖。
3. 幅式选择。
4. 章法预览。
5. 保存草稿。

### 作品页

核心职责：管理草稿和导出结果。

区域顺序：

1. 草稿列表。
2. 最近导出。
3. 导出操作。

## 6. 今日页详细设计

### 6.1 今日任务头部

目标：给用户一个明确任务，而不是欢迎语。

内容：

- 标题：`今日临摹：永 · 欧体`
- 副标题：`看帖 → 拆笔画 → 练结构 → 创章法`
- 今日字：永 / 山 / 水 / 月等系统推荐。
- 当前书体：欧体 / 颜体等。

交互：

- 换一个字。
- 切换书体。

文案约束：

- 不使用“先临摹一个参考字，再拆笔画和结构”这类产品说明式文案。
- 标题必须任务化，让用户一眼知道今天练什么。
- 副标题只做路径提示，字重和颜色要弱于标题。

### 6.2 临摹参考主卡

目标：成为页面绝对视觉中心。

内容：

- 大号参考字。
- 米字格或九宫格。
- 书体、书家、碑帖来源。
- 临摹模板入口：米字格、九宫格、双钩。
- 主要按钮：开始临摹 / 记录一次临摹。

视觉要求：

- 移动端参考字区域占屏幕高度 35% 到 40%。
- 桌面端临摹参考大卡占主内容区 60% 宽度。
- 参考字字号使用动态适配：`clamp(80px, 20vw, 200px)`。
- 参考字面积至少占主卡 40%，理想值为 50% 到 60%。
- 字体展示区不与其他卡片平级，必须是主视觉。
- 不使用花哨背景，不做营销化 hero。
- 使用宣纸感的浅色底，但避免过度仿古。
- 不显示内部 ID。`common-practice-ou` 必须映射成用户可读来源，例如“参考范字”或“出自《九成宫》”。
- 书体信息显示为“欧体 · 欧阳询”，来源另起小字显示。

模式切换：

- “米字格临摹 / 九宫格结构 / 双钩练习”不能是普通标签。
- 必须改为单选切换按钮组。
- 默认选中“米字格临摹”。
- 切换后参考字背景实时变化：米字格、九宫格、无格/双钩。
- MVP 如果暂不实现真实双钩，双钩模式可以显示“轮廓参考待接入”，但按钮状态必须可见。

主操作：

- “记录一次临摹”是今日页核心动作，必须是实心主按钮。
- 按钮放在参考字正下方或主卡右侧首位。
- 点击后显示“已记录 ✓”，今日练习次数立即 +1。

### 6.3 基本笔画模块

目标：让用户知道这个字该从哪些笔法看。

内容：

- 只展示当前字实际包含的笔画。
- 例如“永”不应泛泛展示“横、竖、撇、捺、点、折、钩”，而应展示“点、横折钩、横撇、撇、捺”。
- 每种笔画显示关键提示：
  - 起笔。
  - 行笔。
  - 收笔。
- 代表字入口：一、十、人、永、心。

交互：

- 点击笔画后，高亮参考字中的对应笔画。
- 如果 MVP 暂时无法高亮真实笔画，必须至少更新说明，例如：“永字首笔为点，起笔藏锋，收笔轻提。”
- 选中的笔画必须有明显选中态。

### 6.4 单字结构模块

目标：让用户理解间架结构。

内容：

- 中宫。
- 主笔。
- 重心。
- 收放。
- 避让。
- 穿插。

交互：

- 点击结构标签，在参考字上显示辅助线。
- MVP 必须至少绘制静态辅助线：中宫框、重心线、主笔方向箭头三类之一。
- 点击“中宫 / 主笔 / 重心 / 收放”后，示意图或说明必须变化。
- 纯文字解释不满足验收。

### 6.5 多字章法模块

目标：把单字练习自然引到作品创作。

内容：

- 当前推荐幅式。
- 简短章法示例，例如“山高月小 / 水落石出”。
- 行距、字距、留白提示。
- 入口：用这个书体生成章法。
- 模块内必须直接显示一个章法缩略图，不允许只显示文字。

交互：

- 点击进入创作页，并继承当前书体。
- 后续可继承今日字或推荐诗句。
- 缩略图应使用当前幅式比例，例如斗方为 1:1，条幅为窄长比例。

### 6.6 推荐字队列

目标：给用户下一步练习，不抢主视觉。

内容：

- 永、山、水、月、人、心、中、和。
- 按难度或主题分组。

交互：

- 点击后刷新临摹参考主卡。
- 推荐字保持一行优先展示，行尾放“更多 →”进入查字页。
- 不要把“查更多字”做成大按钮抢占学习路径。

### 6.7 学习进度摘要

目标：反馈用户投入，但不喧宾夺主。

内容：

- 今日已练次数。
- 连续天数。

位置：今日页底部或侧栏，不应放在首屏最前。

约束：

- 新用户首屏不直接突出“累计练习 0”。
- 收藏字不放在今日页首屏，后续放入个人中心或作品/收藏页。

## 7. 查字页详细设计

查字页要像“书法字典”，但服务临摹。

必须包含：

- 单字搜索。
- 书体筛选。
- 候选字卡片。
- 来源信息。
- 进入临摹按钮。

候选字卡片不应只显示一个字符，还应显示：

- 书体。
- 书家。
- 碑帖。
- 是否已练习。

单字详情应复用“临摹参考主卡”的设计，避免今日页和查字页体验割裂。

详情页规范：

- 标题使用“永 · 欧体（欧阳询）”。
- 来源使用小字：“出自《九成宫》”或“参考范字”。
- 不重复使用“临摹参考字”作为大标题。
- 结构要点和笔法要点必须配图示或辅助线。
- 点击“记录一次临摹”后，应将该字设为今日字，并提示“已设为今日练习”或跳转今日页。

## 8. 创作页详细设计

创作页要像“章法工作台”，不是普通表单。

必须包含：

- 文本输入。
- 书体选择。
- 幅式选择。
- 章法预览。
- 保存草稿。
- 导出参考。

章法预览必须强化：

- 纸张比例。
- 行列方向。
- 字距。
- 行距。
- 留白。
- 落款位置。

MVP 可以继续使用 SVG 预览，但视觉上应更像纸面，不像调试图。

空状态规范：

- 未生成前展示虚线纸张框，而不是普通空状态文字。
- 虚线框比例跟随当前幅式。
- 框内浅色提示：“输入内容后点击生成章法”。

幅式选择规范：

- 不使用纯下拉文本作为唯一入口。
- 优先用小卡片展示比例：斗方、中堂、条幅、扇面。
- MVP 只支持斗方时，其他幅式可灰显，但必须让用户知道能力边界。

按钮权重：

- “生成章法”为主操作，使用实心主按钮。
- “保存草稿”为次操作，使用描边按钮。
- 生成后在预览区直接渲染，不弹窗、不跳页。
- 保存后显示“已保存”，并提供去作品页入口。

## 8.1 作品页详细设计

作品页不是冷冰冰的文件列表。它应承接创作成果。

空状态：

- 显示空画框或纸张图标。
- 文案：“你的第一幅作品将在这里出现”。
- 提供按钮：“去创作”。

有内容时：

- 展示作品缩略图。
- 显示标题、书体、幅式、更新时间。
- 操作：继续编辑、导出、删除。

## 9. 视觉规范

### 9.1 整体风格

关键词：

- 安静。
- 专注。
- 类字帖。
- 教学工具。
- 适合长时间学习。

避免：

- 营销首页。
- 大面积渐变。
- 过多卡片堆叠。
- 彩色标签泛滥。
- 统计信息抢主视觉。

### 9.2 色彩

建议：

- 背景：`#F7F4EB` 或 `#F5F0E6`。
- 卡片：`#FFFFFF`。
- 卡片阴影：`0 2px 8px rgba(0,0,0,0.04)`。
- 主色：墨色、深绿或低饱和棕。
- 辅助线：浅赭色。
- 强调色：只用于主按钮。

### 9.3 字体与字号

参考字要大，说明文字要克制。

- 参考字：主视觉最大。
- 参考字字体栈：`"华文楷体", "KaiTi", "STKaiti", serif`。
- 参考字字号：`clamp(80px, 20vw, 200px)`。
- 页面标题：中等，不抢参考字。
- 标签和说明：紧凑。
- 卡片标题：清晰但不夸张。

### 9.4 布局

桌面端建议两栏：

- 左侧：临摹参考和今日任务。
- 右侧：笔画、结构、章法训练路径。

移动端建议单列：

1. 今日任务。
2. 临摹参考。
3. 基本笔画。
4. 单字结构。
5. 多字章法。
6. 推荐字。

底部导航：

- “集字创作”统一改为“创作”。
- 当前选中态必须明显：墨色填充图标、文字加粗或选中背景。
- 今日页为默认入口。

- 常用字查询。
- 大号参考字。
- 米字格。
- 结构说明。
- 笔法说明。

- 练习成功后的下一步建议。

- 临摹照片上传和对比。

## 11. MVP 重构范围

本轮重构只做前端体验和现有数据能力重组，不做大规模底层能力扩展。

包含：

- 重构今日页。
- 复用临摹参考主卡。
- 增加默认今日字加载。
- 调整查字详情。
- 调整章法入口。
- 优化移动端和桌面端布局。
- 增加 widget 测试保障关键入口。

不包含：

- 真实碑帖版权资源接入。
- AI 评分。
- AR 临摹。
- 手写识别。
- 复杂动画笔顺。

## 12. 验收标准

### 12.1 首屏验收

用户登录后，首屏必须能清楚回答：

- 今天练哪个字？
- 参考字在哪里？
- 从哪些笔画看？
- 从哪些结构看？
- 怎么进入章法创作？

### 12.2 功能验收

以下动作必须可用：

- 注册。
- 登录。
- 默认加载今日字。
- 切换常用字。
- 查看临摹参考。
- 查看笔法要点。
- 查看结构要点。
- 记录一次临摹。
- 查更多字。
- 生成章法。
- 保存草稿。
- 导出参考。
- 一键换字。
- 今日任务状态更新。
- 练习后展示下一步建议。
- 错误信息不出现内部异常名。

### 12.3 体验验收

- 临摹参考字必须是页面最明显元素。
- 基本笔画、单字结构、多字章法必须在今日页可见。
- 用户不应依赖底部导航才能知道如何开始学习。
- 统计信息不得抢占主视觉。
- 页面在手机和桌面上都不能出现关键内容被挤出或难以发现的问题。
- 全局背景不能是纯白后台风。
- 参考字不能使用普通黑体作为主要呈现。
- 内部技术 ID 不得暴露给用户。
- 笔画标签必须与当前字相关。
- 结构模块必须有可视化辅助线或示意图。
- 章法模块必须有缩略图。
- 点击学习动作必须有即时反馈。

### 12.4 交互反馈验收

| 操作 | 必须反馈 |
| --- | --- |
| 换一个字 | 参考字更新，有淡入或明显状态变化 |
| 记录一次临摹 | 显示“已记录 ✓”，今日已练 +1 |
| 点击笔画标签 | 标签选中，说明或参考字高亮变化 |
| 点击结构标签 | 辅助线或结构说明变化 |
| 生成章法 | 预览区直接渲染章法图 |
| 保存草稿 | 显示“已保存”，可跳转作品页 |

## 12.5 P0 易用性验收

生产试用前必须优先完成以下 P0 项：

- 新用户登录后能直接看到今日字和 3 步学习路径。
- 今日页有明显的“换一个字”。
- 主按钮文案为“我已临摹”，不再使用“记录一次临摹”。
- 点击“我已临摹”后展示“已记录 ✓”，并出现下一步建议。
- 查字按钮文案改为“查名家写法”。
- 创作按钮文案改为“生成作品布局”。
- 错误提示统一转换为用户可理解文案。

## 13. 后续迭代建议

### 第一阶段：学习路径重构

目标：让当前 MVP 像一个真正学习软件。

工作：

- 今日页重构。
- 默认今日字。
- 临摹参考主卡。
- 三段训练路径。
- 查字详情复用。

### 第二阶段：内容可信度提升

目标：让参考更像真实字帖。

工作：

- 接入授权碑帖切图。
- 为常用字补真实图像。
- 增加来源展示。
- 增加书体对比。

### 第三阶段：练习反馈闭环

目标：让用户练完能得到反馈。

工作：

- 练习历史。
- 每日任务完成状态。
- 薄弱笔画推荐。
- 作品集沉淀。

### 第四阶段：智能能力

目标：让应用像随身书法导师。

工作：

- 拍照对比。
- 结构偏差分析。
- 运笔动画。
- 个性化推荐。

## 14. 结论

当前产品不应继续以“功能页”思路迭代。正确方向是把星云书法做成“临摹训练驱动”的学习工作台。

下一步实现应围绕一句话展开：

> 用户打开应用后，立刻知道今天练什么、怎么看帖、怎么拆笔画结构、怎么进入章法创作。

只有做到这一点，书法应用才会从“能用”走向“好用”。
